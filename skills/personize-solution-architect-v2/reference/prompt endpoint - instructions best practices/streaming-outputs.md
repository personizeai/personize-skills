# Prompt Streaming with Structured Outputs

## Overview

The `/api/v1/prompt` endpoint supports **streaming with structured outputs** — real-time delivery of named output blocks as they are generated by the LLM.

This is critical for use cases like website personalization, where different parts of a page need content delivered progressively rather than waiting for all content to finish generating.

## How It Works

### Request

```typescript
const stream = personize.ai.promptStream({
  context: '...',
  outputs: [
    { name: 'hero' },
    { name: 'social_proof' },
  ],
  instructions: [{
    prompt: 'Generate a hero section and social proof...',
    maxSteps: 3,
  }],
});
```

### Server-Side Processing

1. LLM generates text containing `<output name="hero">...</output>` XML markers
2. The `OutputStreamParser` (state machine) parses markers in real-time as chunks arrive
3. When `</output>` closes for a named output, the server immediately emits an SSE event
4. After all content is generated, a final `done` event includes all outputs for reconciliation

### SSE Event Types

| Event | When | Payload |
|---|---|---|
| `text` | Every text chunk outside markers | `{ type: "text", chunk: "..." }` |
| `output` | Each time `</output>` closes | `{ type: "output", name: "hero", data: {...} }` |
| `step_complete` | After each instruction step | `{ type: "step_complete", instructionIndex: 0, ... }` |
| `done` | End of stream | `{ type: "done", outputs: {...}, metadata: {...} }` |
| `error` | Non-fatal error | `{ type: "error", message: "..." }` |

### Output Data

The `data` field in `output` events is:
- **Parsed JSON** if the content between markers is valid JSON
- **Raw string** if not valid JSON

No schema validation is performed — the LLM decides what shape each output takes.

## Structured Outputs for Website Personalization (gs-edge)

### Zone Types

```
collection:property  →  Memory lookup (instant)     e.g., "Contacts:firstName"
output.field         →  AI output JSON field         e.g., "hero.headline"
plain-id             →  AI output plain text          e.g., "tagline"
```

### Fan-Out Pattern

When generating content for a page with grouped zones like `hero.headline`, `hero.subtitle`, `hero.cta`:

1. **Group by output name**: `hero → [headline, subtitle, cta]`
2. **Send one output per group**: `outputs: [{ name: "hero" }]`
3. **Instruct LLM to return JSON**: `"hero" must be JSON with keys: headline, subtitle, cta`
4. **Fan out on arrival**: When `output` event arrives for `hero`, extract each field and deliver as separate zone events

This reduces LLM overhead (fewer XML markers) and ensures semantic coherence within each group.

### Prompt Instruction Template

```
Generate personalized website content for the following outputs.

OUTPUTS:
- "hero": JSON object with these keys:
    "headline": (plain text string)
    "subtitle": (plain text string)
    "cta": (plain text string)
- "tagline": plain text string — Write a compelling tagline

RULES:
- Structured outputs MUST be valid JSON with exactly the specified keys
- Flat outputs MUST be plain text, no JSON wrapping
- Output ONLY plain text per field. No HTML, no markdown.
- NEVER invent facts not in the context.
- Use "SKIP" for fields you lack data for.
```

## SDK Usage

### Async Generator (Node.js)

```typescript
for await (const event of personize.ai.promptStream({
  prompt: '...',
  outputs: [{ name: 'result' }],
})) {
  if (event.type === 'output') {
    console.log(event.name, event.data);
  }
  if (event.type === 'done') {
    break;
  }
}
```

### Raw Fetch (when SDK is not available)

```typescript
const response = await fetch(`${baseURL}/api/v1/prompt`, {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${secretKey}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    stream: true,
    outputs: [{ name: 'result' }],
    instructions: [{ prompt: '...' }],
  }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();
let buffer = '';

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buffer += decoder.decode(value, { stream: true });

  const parts = buffer.split('\n\n');
  buffer = parts.pop() || '';

  for (const part of parts) {
    for (const line of part.split('\n')) {
      if (line.startsWith('data: ')) {
        const event = JSON.parse(line.slice(6));
        // Handle event...
      }
    }
  }
}
```

## Key Considerations

- **Timeout**: Default stream timeout is 120s in SDK, configurable via `streamTimeout`
- **Cancellation**: Pass `signal: AbortSignal` for external cancellation
- **Fallback**: If streaming fails, fall back to sync `prompt()` which returns all outputs at once
- **No schema validation**: Outputs are AI-generated — always handle missing fields gracefully
- **Auto-memorize**: Works with streaming — `memorize` config in request auto-saves extracted outputs
